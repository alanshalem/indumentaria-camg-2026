📋 Especificación de Proyecto: Sistema de Pedidos de Indumentaria - Club Atlético Monte Grande
🎯 Objetivo
Crear una aplicación web para que los socios del Club Atlético Monte Grande realicen pedidos de indumentaria deportiva sin necesidad de backend complejo.

🛠 Stack Tecnológico
Frontend

React 18+ con Vite
CSS Modules para estilos aislados
React Router para navegación
Zustand o Context API para state management
LocalStorage para persistencia de pedidos

Backend (Opcional/Mínimo)

Vercel Serverless Functions (si es necesario para CAPTCHA o almacenamiento)
Alternativamente: Todo en cliente con localStorage/indexedDB

Hosting

Vercel (deployment directo desde Git)


🎨 Identidad Visual
Colores del Club

Rojo principal: #DC143C o similar (extraer del Instagram)
Negro: #1a1a1a
Blanco: #ffffff
Grises: para backgrounds y textos secundarios

Referencia

Instagram: @clubatleticomontegrande
Usar logos e imágenes oficiales del club


📦 Funcionalidades Principales
🔴 Para los Socios (Página Pública)
1. Catálogo de Productos
- Mostrar productos de indumentaria con:
  ✓ Imagen del producto
  ✓ Nombre/descripción
  ✓ Talles disponibles (S, M, L, XL, XXL, etc.)
  ✓ Colores disponibles (chips visuales)
  ✓ Botón "Agregar al carrito"
2. Carrito de Compras
- Vista lateral/modal con:
  ✓ Lista de productos agregados
  ✓ Talle y color seleccionado por producto
  ✓ Cantidad de cada item
  ✓ Botón para eliminar items
  ✓ Total de productos
3. Checkout / Generación de Pedido
Formulario simple:
  ✓ Nombre
  ✓ Apellido
  ✓ Botón "Generar Pedido"

Al presionar:
  1. Generar código único (ej: "CAMG-2024-00123")
  2. Guardar pedido en localStorage con:
     - Código
     - Fecha y hora
     - Nombre y apellido
     - Items del carrito
  3. Mostrar pantalla de confirmación con el código
  4. Guardar código en cookie/localStorage con duración máxima
4. Recuperación de Código
- Al volver a ingresar:
  ✓ Detectar si hay código guardado
  ✓ Mostrar banner/modal con el código del último pedido
  ✓ Indicar fecha del pedido
  ✓ Permitir hacer un nuevo pedido sin perder el anterior

⚫ Para Administradores (Dashboard Oculto)
1. Autenticación /login
- URL oculta: /login
- Clave maestra (hardcodeada o en env)
- CAPTCHA simple (puede ser:
  ✓ hCaptcha
  ✓ reCAPTCHA
  ✓ Un simple "cuánto es 5+3?" si no quieres servicios externos)
- Session temporal guardada en sessionStorage
2. Dashboard de Pedidos
Vista de tabla/lista con:
  ✓ Código de pedido
  ✓ Fecha y hora
  ✓ Nombre y apellido del socio
  ✓ Detalle de productos (expandible)
  ✓ Estado (opcional: pendiente/entregado)
  ✓ Filtros por fecha/nombre
  ✓ Búsqueda por código

Item del Carrito
typescriptinterface CartItem {
  productId: string;
  productName: string;
  size: string;
  color: string;
  quantity: number;
}
Pedido
typescriptinterface Order {
  code: string; // "CAMG-2024-00123"
  timestamp: number;
  customerName: string;
  customerLastName: string;
  items: CartItem[];
  status?: "pending" | "delivered"; // Opcional
}

LocalStorage

localStorage.setItem('camg_last_order_date', Date.now());

📝 Notas Importantes
⚠️ Limitaciones sin Backend
✓ Los pedidos solo existen en localStorage del navegador
✓ Si el admin limpia el navegador, pierde los datos
✓ No hay sincronización entre dispositivos
✓ Para persistencia real, considerar:
  - Vercel Postgres (free tier)
  - Supabase (gratuito hasta cierto punto)
  - Google Sheets API (hacky pero funcional)
